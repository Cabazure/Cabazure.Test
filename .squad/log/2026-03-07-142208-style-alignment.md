### Field Naming Convention

**Decision:** Private instance fields and private static fields use **camelCase with no prefix** (no `_`, no `s_`).

**Affected Files:**
- `src/Cabazure.Test/Fixture/SutFixture.cs` — `_fixture` → `fixture`
- `src/Cabazure.Test/Customizations/SutFixtureCustomizations.cs` — `_customizations` → `customizations`, `_lock` → `syncLock`

**Rationale:**
- Aligns with Cabazure sibling repos (organization-wide consistency)
- Matches .NET design guidelines for private members (BCL preference for camelCase)
- Enforced via `.editorconfig` analyzer rules at editor/CI level
- Avoids accidental shadowing confusion; reviewers use `this.` qualification when needed (rare)

**Special Case:** `syncLock` (not `lock`) because `lock` is a reserved C# keyword. This is the correct camelCase name, not an exception.
